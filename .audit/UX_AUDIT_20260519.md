# UI/UX Audit — local dev, 2026-05-19

Ran against `npm run dev` (`localhost:3000`) with stub Supabase
(demo mode). Tested at desktop (1280–1440) and mobile (375×812).
Storefront + admin, both sessions (anon + owner).

## What got fixed — by severity

### P0 — broken (page-level)

| # | Issue | Resolution |
|---|---|---|
| 1 | Async server component (`SearchOverlayWrapper`) imported into a `'use client'` tree (`SiteChrome`). Console flooded with "uncached promise"; pages suspended past the hero. | Moved data fetch to `app/layout.tsx`; pass `searchTrending`/`searchCategories` as props through SiteChrome. Deleted the wrapper file. |
| 2 | `AddToCartToast` fired on every reload that had a saved cart — localStorage hydration moved `cartCount` from 0 → N and the effect treated it as an add. | Added `addCounter` + `lastAdded` to `CartContext` (only set by explicit `addToCart`). Toast subscribes to those, not to `cartCount`. |
| 3 | `useBodyScrollLock` left body in `overflow:hidden + position:fixed` after Fast Refresh or two overlapping modals (mini-cart + a popover). Page un-scrollable for the rest of the session. | Refactored to ref-counted via `body.dataset.ypScrollLockCount`. First lock records `scrollY`; last release restores. Survives HMR. Was the original bug behind "opened menu and the page below kept scrolling". |

### P1 — visibly wrong

| # | Issue | Resolution |
|---|---|---|
| 4 | Footer newsletter form overflowed its 200 px column on every page. | Added `minWidth:0` to the flex input so the browser's 20-char content min stops fighting the layout. |
| 5 | Category tiles bottom labels at 10 px white-on-image; product-tile brand line at 11 px. | Bumped category labels to 13 px, brand line to 12 px, tightened letter-spacing. |
| 6 | Brand pink `#E8487F` failed WCAG 1.4.3 (3.4:1) with white text on every CTA: Shop Beauty, Add to Cart, Checkout, Subscribe, header cart-count badge, review avatar, PWA install, error pages, addresses default-pill. | Added `--brand-pink-cta: #C5286A` (5.4:1). Migrated every fill-with-white-text usage. Decorative pink (chips, hearts, dots) kept the lighter tone. |
| 7 | Mobile `PromoBanner` ate ~250 px of the first viewport (label + headline + subline + CTA stacked). | Tightened mobile padding; dropped the subline below 768 px; shrunk the headline. Desktop still gets the full editorial card. |
| 8 | Admin sidebar rendered the X close button on desktop. | Hidden via CSS `display:none !important` at ≥768 px; re-shown on mobile. |
| 9 | 28 admin files hard-coded Tailwind pink-500 `#ec4899` — same WCAG failure as the storefront brand pink. | Bulk-swapped to `#C5286A` across `/admin/**`. |
| 10 | `/admin/settings` destructured `searchParams.saved` synchronously — Next 16 made it a Promise. | Awaited alongside `getSiteSettings()`. |
| 11 | AdminShell read `window.matchMedia` inside JSX to set `aria-hidden` → hydration mismatch on every mobile load. | Replaced with `useState` + a `matchMedia('change')` listener post-mount. |
| 12 | CategoryTiles label gradient (42 % black bottom) didn't darken enough over light thumbnails → ~2.4:1. | Bumped to 62 % — all eight tile labels clear AA. |

### P2 — polish

| # | Issue | Resolution |
|---|---|---|
| 13 | Search overlay placeholder "Search products, brands, concerns…" at 18 px overflowed the 238 px mobile input — Close button got cropped. | Shortened to "Search products…"; added `minWidth:0` for flex shrink. |
| 14 | MiniCart per-line brand at 9 px (0.5625 rem). | Bumped to 11 px (0.6875 rem) — matches the default Overline tier. |
